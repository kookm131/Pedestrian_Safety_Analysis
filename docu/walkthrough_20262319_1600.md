# Walkthrough - Upload Failure Resolved

동영상 업로드 시 발생하던 '500 Internal Server Error' 문제를 해결했습니다.

## 원인 분석
- **문제**: [docker-compose.yml](file:///c:/work/tech1/docker-compose.yml) 파일에서 `backend` 서비스의 `MONGO_URL` 환경 변수가 잘못된 포트(`27117`)를 사용하고 있었습니다.
- **영향**: 백엔드가 업로드된 파일을 MongoDB GridFS에 저장하려고 시도할 때 연결 오류가 발생하여 500 에러를 반환했습니다.

## 해결 방법
- [docker-compose.yml](file:///c:/work/tech1/docker-compose.yml)에서 `backend` 서비스의 `MONGO_URL` 포트를 표준 포트인 `27017`로 수정했습니다.
- 서비스를 재시작하여 설정을 적용했습니다.

## 검증 결과
1. **백엔드 업로드 테스트**: `curl`을 이용한 업로드 테스트 시 `200 OK`와 함께 `file_id`가 정상적으로 반환됨을 확인했습니다.
2. **전체 파이프라인 작동 확인**: 
   - 업로드된 파일이 MongoDB GridFS에 저장됨.
   - RabbitMQ를 통해 분석 워커에 메시지가 전달됨.
   - 워커가 이를 수신하여 분석(시뮬레이션) 후 결과를 **PostgreSQL**에 저장함.
3. **데이터베이스 확인**: PostgreSQL의 `analysis_results` 테이블에 분석 데이터가 정상적으로 입력된 것을 확인했습니다.

이제 웹 인터페이스에서 동영상을 업로드하면 정상적으로 분석이 진행되고 결과가 표시됩니다.
