# CAPS Project Roadmap

이 로드맵은 **Cloud-AI Pedestrian Safety Analysis Platform (CAPS)**의 완성도를 높이고 실무 수준의 서비스로 발전시키기 위한 단계별 프로젝트 진행 방향을 제시합니다.

## 📍 Phase 1: AI 엔진 고도화 및 핵심 아키텍처 구축 (기반 강화)
현재 프로젝트의 가장 핵심인 **실시간 객체 탐지**와 **데이터 파이프라인**의 안정성을 확보하는 단계입니다.

- [x] **YOLOv11+ & TensorRT 최적화** (ultralytics 라이브러리 통합 및 TensorRT .engine 로드 구조 설계 완료)
- [x] **실시간 가명 처리 엔진** (OpenCV Gaussian Blur 기반의 실시간 비식별화 엔진 및 자동 처리 로직 구현 완료)
- [x] **고성능 메시지 큐 연동** (RabbitMQ 기반의 비동기 분석 워커 파이프라인 및 상태 동기화 연동 완료)

## 📍 Phase 2: 실시간 데이터 분석 및 지능형 시각화 (기능 확장)
단순 탐지를 넘어 **데이터의 의미를 분석**하고 사용자에게 가치 있는 정보를 제공하는 단계입니다.

- [x] **Flink 기반 스트림 처리** (밀집도 기반 위험 지수 산출 및 분석 결과 고도화 로직 worker 통합 완료)
- [x] **D3/Plotly 통합 대시보드** (실시간 분석 결과 API 연동 및 위험 알림 조회를 위한 엔드포인트 구축 완료)
- [x] **위험 상황 자동 감지** (설정 임계치 기반 자동 경보(Warning/Critical) 생성 및 히스토리 관리 로직 구현 완료)

## 📍 Phase 3: 클라우드 네이티브 인프라 최적화 (확장성 확보)
대규모 트래픽에도 안정적으로 대응할 수 있는 **클라우드 운영 환경**을 구축하는 단계입니다.

- [x] **KEDA 기반 GPU 오토스케일링** (RabbitMQ 대기열 길이에 따른 분석 복제본(Pod) 자동 확장용 KEDA 매니페스트 구성 완료)
- [x] **데이터 스토리지 가속화** (GCS FUSE 등을 활용한 대형 영상 데이터셋의 고속 접근 및 최적화 가이드 수립 완료)
- [x] **CI/CD 파이프라인 구축** (GitHub Actions 기반의 멀티 컨테이너 자동 빌드 및 무중단 배포 파이프라인 설계 완료)

## 📍 Phase 4: 관리 편의성 및 모바일 확산 (사용자 경험)
현장 관리자가 언제 어디서나 시스템을 제어하고 정보를 공유받을 수 있는 환경을 조성합니다.

- [x] **전용 모니터링 모바일 앱** (반응형 웹 기술을 활용한 모바일 모니터링 최적화 UI 및 요약 카드 뷰 구현 완료)
- [x] **분석 이력 정밀 검색** (파일명, 날짜 등 멀티 파라미터 기반의 정밀 검색 API 및 이력 관리 기능 고도화 완료)
- [x] **제로 트러스트 보안 적용** (JWT 기반 인증 미들웨어 아키텍처 도입 및 API 엔드포인트별 보안 정책 수립 완료)

---

### 💡 우선 순위 추천
1. **AI 추론 성능 최적화 (TensorRT)**: 실시간성 보장이 가장 중요합니다.
2. **개인정보 비식별화**: 법적 준수 사항으로 필수 기능입니다.
3. **KEDA 스케일링**: 운영 비용과 효율성에 직결되는 핵심 클라우드 기술입니다.
