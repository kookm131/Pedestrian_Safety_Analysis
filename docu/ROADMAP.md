# CAPS Project Roadmap

이 로드맵은 **Cloud-AI Pedestrian Safety Analysis Platform (CAPS)**의 완성도를 높이고 실무 수준의 서비스로 발전시키기 위한 단계별 프로젝트 진행 방향을 제시합니다.

## 📍 Phase 1: AI 엔진 고도화 및 핵심 아키텍처 구축 (기반 강화)
현재 프로젝트의 가장 핵심인 **실시간 객체 탐지**와 **데이터 파이프라인**의 안정성을 확보하는 단계입니다.

- [ ] **YOLOv11+ & TensorRT 최적화**: 최신 알고리즘을 도입하고 NVIDIA TensorRT를 적용하여 엣지/클라우드 환경에서의 추론 속도(Latency)를 극대화합니다.
- [ ] **실시간 가명 처리 엔진**: 영상 수집 즉시 안면 및 차량 번호판을 탐지하여 개인정보를 블러링하는 로직을 분석 파이프라인 최우선 순위에 배치합니다.
- [ ] **고성능 메시지 큐 연동**: RabbitMQ를 통해 대량의 영상 프레임 데이터를 백엔드와 AI 서버 간에 지연 없이 전달하는 버퍼 구조를 완성합니다.

## 📍 Phase 2: 실시간 데이터 분석 및 지능형 시각화 (기능 확장)
단순 탐지를 넘어 **데이터의 의미를 분석**하고 사용자에게 가치 있는 정보를 제공하는 단계입니다.

- [ ] **Flink 기반 스트림 처리**: 실시간으로 유입되는 탐지 로그를 Apache Flink로 처리하여 구역별 밀집도 지수 및 위험 지표를 실시간으로 계산합니다.
- [ ] **D3/Plotly 통합 대시보드**: 누적된 데이터를 바탕으로 시간대별 유동인구 추이, 위험 지역 히트맵(Heatmap) 등 시각화 컴포넌트를 고도화합니다.
- [ ] **위험 상황 자동 감지**: 무단횡단, 갑작스러운 인파 밀집, 낙상 사고 등 특정 시나리오를 감지하고 즉시 알람을 발생시키는 이벤트를 정의합니다.

## 📍 Phase 3: 클라우드 네이티브 인프라 최적화 (확장성 확보)
대규모 트래픽에도 안정적으로 대응할 수 있는 **클라우드 운영 환경**을 구축하는 단계입니다.

- [ ] **KEDA 기반 GPU 오토스케일링**: RabbitMQ의 큐 적체량에 따라 분석 서버(GPU Pod)를 자동으로 확장/축소하여 클라우드 비용을 절감하고 처리량을 보장합니다.
- [ ] **데이터 스토리지 가속화**: GCS FUSE 또는 S3 분산 스토리지를 활용하여 대규모 비디오 데이터셋에 대한 학습 및 분석 접근 속도를 최적화합니다.
- [ ] **CI/CD 파이프라인 구축**: 모델 업데이트 시 자동으로 컨테이너를 빌드하고 무중단 배포(Blue-Green 또는 Canary)가 가능하도록 자동화합니다.

## 📍 Phase 4: 관리 편의성 및 모바일 확산 (사용자 경험)
현장 관리자가 언제 어디서나 시스템을 제어하고 정보를 공유받을 수 있는 환경을 조성합니다.

- [ ] **전용 모니터링 모바일 앱**: 웹 대시보드의 기능을 축약하여 관리자가 이동 중에도 긴급 알림을 받고 현장을 확인할 수 있는 앱 UI를 개발합니다.
- [ ] **분석 이력 정밀 검색**: 저장된 영상 데이터와 AI 로그를 결합하여 특정 위치/시간대의 이벤트를 빠르게 검색하고 리포트를 생성하는 기능을 추가합니다.
- [ ] **제로 트러스트 보안 적용**: 외부 접속 시 데이터 권한 관리를 강화하고 전 구간 암호화를 통해 보안 완성도를 높입니다.

---

### 💡 우선 순위 추천
1. **AI 추론 성능 최적화 (TensorRT)**: 실시간성 보장이 가장 중요합니다.
2. **개인정보 비식별화**: 법적 준수 사항으로 필수 기능입니다.
3. **KEDA 스케일링**: 운영 비용과 효율성에 직결되는 핵심 클라우드 기술입니다.
